## üß© Functional Requirements

### 1. Authentication & Attempt Control

* Google Authentication only
* One Google account = one quiz attempt (hard lock)
* Attempt state must persist across refresh, reconnect, or accidental close
* No admin overrides, no reset UI
* Once submitted or terminated ‚Üí permanently locked

---

### 2. JSON-Driven Quiz Engine (No Admin Panel)

* Quiz questions are loaded from a **static or remote JSON file**
* JSON schema supports:

  * Question ID
  * Type:

    * MCQ (single correct)
    * MCQ (multiple correct)
    * True / False
    * Code output (optional)
  * Options
  * Correct answer(s)
  * Difficulty (Easy / Medium / Hard)
  * Optional explanation
* Validate JSON at runtime
* Graceful fallback if JSON is malformed

---

### 3. Timed Quiz Flow

* Timer starts only after user clicks **‚ÄúStart Quiz‚Äù**
* Sticky floating timer with subtle animation
* Auto-submit on timeout
* Confirmation modal before manual submit
* Save answers incrementally (every change)

---

### 4. Anti-Cheat (Soft but Effective)

* Randomized question order per user
* Randomized option order per question
* Detect tab switch / window blur:

  * First 2 times ‚Üí animated warning toast
  * 3rd time ‚Üí auto-submit + lock
* Disable copy/paste where feasible
* Prevent reload abuse using persisted state

---

### 5. Evaluation & Scoring

* Auto-evaluate on submit or timeout
* Score formula:

  * Accuracy
  * Time taken (weighted)
* Tie-breakers:

  1. Higher score
  2. Higher accuracy
  3. Lower time taken
* Evaluation must be deterministic and reproducible

---

### 6. Leaderboard

* Auto-generated from attempts
* Display:

  * Rank
  * Name
  * Score
  * Accuracy %
  * Time taken
* Smooth animated rank transitions
* Final leaderboard freezes after quiz end
